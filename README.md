# Apple iPhone (marketing site)

By **Manya Shukla**.

Single-page React experience inspired by Apple’s iPhone product pages: hero, highlights, interactive 3D model (React Three Fiber), features, and footer. Built with **Vite**, **React 18**, **Tailwind CSS**, **GSAP**, and **Three.js** (via `@react-three/fiber` and `@react-three/drei`). Error monitoring is wired for **Sentry** in `vite.config.js` (optional for local dev).

## Prerequisites

- **Node.js** (LTS recommended)
- **npm** (this repo uses `package-lock.json`)

## Scripts

| Command        | Description                          |
| -------------- | ------------------------------------ |
| `npm install`  | Install dependencies                 |
| `npm run dev`  | Start Vite dev server (default port) |
| `npm run build`| Production build to `dist/`          |
| `npm run preview` | Serve the production build locally |
| `npm run lint` | Run ESLint on the project            |

After `npm run dev`, open the URL printed in the terminal (typically [http://localhost:5173](http://localhost:5173)).

### Root files (quick reference)

- **`index.html`** — Entry HTML; loads `/src/main.jsx` as an ES module.
- **`vite.config.js`** — Bundler config: `@vitejs/plugin-react`, `@sentry/vite-plugin`, `build.sourcemap`.
- **`tailwind.config.js`** / **`postcss.config.js`** — Styling pipeline for utility classes.

### `public/`

Files here are served from the **site root** in dev and build. The app references many of these via **`/assets/...`** in `src/utils/index.js` (not `public/assets` in the URL). The GLB model lives at **`/models/scene.glb`**.

### `src/components/`

Each file is a React component for one part of the landing page. **`Model`**, **`ModelView`**, **`IPhone`**, **`Lights`**, and **`Loader`** work together for the interactive 3D iPhone; the rest are mostly layout, video, and marketing sections.

### `src/constants/`

Structured content (navigation items, highlight slide definitions, feature text) kept separate from JSX so the page is easier to edit.

### `src/utils/`

**`index.js`** centralizes imports from **`public/assets`** so components and constants use consistent paths. **`animations.js`** holds shared animation logic (GSAP) used across sections.
